# Security

## Reporting a vulnerability

Please **do not** open a public GitHub issue for security problems.

Contact CirriusImpact support through your usual support channel, or open a private advisory if your organization has a security contact for Netech/CirriusImpact.

## Safe issue reporting

When filing public bugs, redact:

- Patron barcodes, names, phone numbers, email
- SFTP/API credentials
- Internal hostnames you do not intend to publish
